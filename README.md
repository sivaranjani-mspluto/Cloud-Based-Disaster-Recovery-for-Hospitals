# Cloud-Based Disaster Recovery for Hospitals

## Overview
This project focuses on implementing a cloud-based disaster recovery solution for healthcare facilities, aligning with UN Sustainable Development Goal 3 (Good Health and Well-being). The system ensures continuous access to critical healthcare data during disasters through cloud infrastructure.

## Features
- Secure cloud backup of hospital data
- Disaster recovery planning and simulation
- Data redundancy and high availability
- Real-time monitoring and alerts
- Compliance with healthcare data regulations

## Technologies Used
- Cloud Provider: AWS/Azure/GCP
- Database: MongoDB Atlas (for NoSQL data)
- Backend: Python (FastAPI)
- Frontend: React.js
- Infrastructure as Code: Terraform

## Setup
1. Clone the repository
2. Install dependencies: `pip install -r requirements.txt`
3. Configure environment variables (copy .env.example to .env)
4. Run the application: `python app.py`

## Dataset
This project utilizes hospital emergency response and patient care data to model disaster scenarios and recovery processes.
